## 1. 系统概述

RAG写作系统是一个基于检索增强生成（RAG）的智能写作平台，默认集成**SkillAgent策略智能体**，能够：

- 从Milvus向量数据库检索相关知识素材
- 通过策略智能体生成结构化的写作策略蓝图
- 基于蓝图编译大纲，再生成高质量文章
- 支持Self-Reflection校验、Few-shot案例学习、蓝图版本回退

### 核心组件

| 组件 | 说明 |
|------|------|
| RAGWriter | 主编排引擎，负责检索和写作 |
| SkillAgent | 策略智能体，生成写作策略蓝图（默认启用） |
| Milvus | 向量数据库，存储文章向量索引 |
| Flask Web | 前端界面服务 |

## 3. 服务启动

### 3.1 启动顺序

**步骤1：启动Milvus服务**

```bash
# 使用Docker启动Milvus
docker run -d --name milvus-standalone \
  -p 19530:19530 \
  -p 9091:9091 \
  -v ./milvus_data:/var/lib/milvus \
  milvusdb/milvus:v2.3.3
```

**步骤2：启动RAG写作系统Web服务**

推荐使用 PM2 进程管理器（支持守护、自动重启、日志管理）：

```bash
cd E:\行业稿件写作\code

# 安装 PM2（需要 Node.js，首次使用执行一次即可）
npm install -g pm2

# 启动服务
pm2 start ecosystem.config.json

# 常用管理命令
pm2 restart rag-writer   # 重启服务
pm2 stop rag-writer      # 停止服务
pm2 status               # 查看运行状态
pm2 logs rag-writer      # 查看实时日志
pm2 delete rag-writer    # 移除服务
```

> 项目根目录提供了便捷脚本：双击 `start.bat` 启动、`restart.bat` 重启、`stop.bat` 停止、`status.bat` 查看状态、`logs.bat` 查看日志。

备用方式（不使用 PM2）：

```bash
cd E:\行业稿件写作\code

# 直接运行（前台，Ctrl+C 停止）
python -m rag_writer.web_demo
```

**步骤3：访问前端**

打开浏览器访问：**http://localhost:5003**

## 4. 系统架构

### 4.1 统一Skill写作流程

系统默认采用SkillAgent策略智能体驱动的写作流程，包含5步向导：

```
用户输入话题
    ↓
RAG检索 → Milvus向量库 + 文本检索
    ↓
头脑风暴（迭代式）
  ├── 检索产品知识库（6大游戏产品）
  ├── 系统生成大角度 → 用户选择 → 系统展开子角度
  ├── 多轮迭代直到用户满意
  └── 选定角度融入后续流程
    ↓
SkillAgent生成策略蓝图
  ├── 检索相似策略案例（Few-shot）
  ├── 融合头脑风暴选定的角度
  ├── 构建策略提示词
  ├── LLM生成JSON蓝图
  └── Self-Reflection校验（可选）
    ↓
审核蓝图（Markdown预览，支持编辑和版本回退）
    ↓
基于蓝图生成大纲（StrategyCompiler编译）
    ↓
审核大纲（可修改）
    ↓
基于大纲写作（write_from_outline）
    ↓
输出文章 + 知识来源 + 蓝图置信度
```

### 4.2 核心设计理念

**策略与事实分离**：
- SkillAgent管"怎么写"（策略层）—— 输出StrategyBlueprint
- RAGWriter管"写什么"（事实层）—— 使用RAG事实素材生成文章

## 5. 使用指南

### 5.1 五步写作流程

系统采用五步向导引导用户完成写作：

**Step 1：填写写作任务**

- 输入话题/主题（必填）
- 填写背景信息、具体要求（可选）
- 上传需求文件（支持TXT/DOCX）
- 设置关键词、字数要求
- 设置RAG检索数量（3-10条）
- 添加补充资料（可选：上传文件、添加链接、粘贴文本）
- 点击"进行头脑风暴"

**Step 2：头脑风暴 - 寻找切入角度**

- 系统自动检索产品知识库，获取相关产品资讯
- AI生成3-5个"大角度"（切入方向和立意）
- 用户点击角度卡片选中感兴趣的，点击"深入展开"
- AI基于选中的角度展开更细致的子角度
- 用户可输入补充想法引导AI展开方向
- 可切换聚焦产品重新生成角度
- 多轮迭代直到找到满意的切入角度
- 可点击"跳过"直接进入蓝图生成

**Step 3：审核策略蓝图**

- 查看基于选定角度生成的策略蓝图（Markdown格式）
- 可提交修改意见让AI修订
- 可手动编辑蓝图文案
- 支持版本回退（可选择回退到历史版本）
- 确认后点击"确认蓝图，生成大纲"

**Step 4：审核大纲**

- 查看基于策略蓝图生成的大纲
- 可提交修改意见让AI修订
- 可手动编辑大纲
- 确认后点击"确认大纲，开始写作"

**Step 5：生成文章**

- 等待文章生成
- 查看生成的文章内容
- 查看知识来源
- 可复制、下载文章、重新生成

### 5.2 上传需求文件

系统支持上传`.txt`或`.docx`文件作为写作需求：

1. 在Step 1点击"上传需求文件"
2. 系统自动提取文件中的话题和要求
3. 自动填充到表单中

### 5.3 补充资料

支持三种补充资料：

| 类型 | 说明 |
|------|------|
| 上传文件 | 支持TXT、DOCX、PDF、Excel |
| 添加链接 | 输入微信文章或其他网页URL |
| 粘贴文本 | 直接粘贴文本内容 |

## 6. API参考

### 6.1 状态检查

```
GET /api/status
```

返回系统状态（Milvus连接、Embedding模型、LLM配置）

### 6.2 头脑风暴

```
POST /api/brainstorm
Content-Type: application/json

{
  "topic": "王者荣耀世界的开放世界设计",
  "background": "王者荣耀世界于2026年上线",
  "requirements": "分析其开放世界设计的创新之处",
  "keywords": ["开放世界", "王者荣耀世界", "游戏设计"],
  "product": null,
  "selected_angle": null,
  "user_feedback": "",
  "brainstorm_history": []
}
```

参数说明：
- `selected_angle`：选中的角度对象（传入则进入"展开模式"，否则为"初始模式"）
- `user_feedback`：用户补充想法，引导展开方向
- `brainstorm_history`：前几轮对话历史

返回：
```json
{
  "angles": [
    {
      "angle_id": "a1",
      "angle_title": "切入角度标题",
      "stance": "核心立意",
      "reasoning": "为什么这个角度有说服力",
      "product_facts": ["事实1", "事实2"],
      "dimension": "design_philosophy",
      "depth": 0
    }
  ],
  "product_context": "相关产品知识摘要...",
  "available_products": [
    {"collection": "honor_of_kings_world", "display_name": "王者荣耀世界", "article_count": 91}
  ]
}
```

## 7. 故障排除

### 7.1 服务无法启动

**问题**：执行`python -m rag_writer.web_demo`报错

**解决**：
```bash
# 1. 检查Python版本
python --version  # 需要 >= 3.9

# 2. 检查依赖
pip install -r requirements.txt

# 3. 检查Milvus服务
docker ps | grep milvus
```

### 7.2 Milvus连接失败

**问题**：`"milvus": {"exists": false}`

**解决**：
```bash
# 1. 启动Milvus
docker start milvus-standalone

# 2. 或重新创建
docker rm milvus-standalone
docker run -d --name milvus-standalone -p 19530:19530 milvusdb/milvus:v2.3.3
```

### 7.3 LLM调用失败

**问题**：生成蓝图/大纲/文章时提示LLM错误

**解决**：
1. 检查`.env`文件中的API Key配置
2. 确认网络能访问LLM服务商
3. 检查API配额

### 7.4 策略蓝图生成失败

**问题**：策略蓝图返回空或格式错误

**解决**：
1. 检查`./strategy_cases`目录是否存在策略案例
2. 检查Milvus中`strategy_cases`集合是否有数据
3. 查看后端日志确认SkillAgent是否正常调用

### 7.5 前端页面显示异常

**问题**：页面样式错乱或JS不工作

**解决**：
1. 清除浏览器缓存
2. 使用Chrome/Firefox最新版本
3. 检查控制台是否有JS错误

### 7.6 查看日志

```bash
# Linux/Mac
python -m rag_writer.web_demo 2>&1 | tee logs.txt

# Windows PowerShell
python -m rag_writer.web_demo 2>&1 > logs.txt
```
