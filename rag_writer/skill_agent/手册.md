# 写作Skill智能体手册

## 一、概述

写作Skill智能体是RAG写作系统的战略升级组件，扮演"策略导师/架构师"角色。它不直接生成文章内容，而是分析用户需求后输出结构化的"写作蓝图"——指导RAG主模型"怎么写"。

### 核心理念

```
策略与事实分离：
├── SkillAgent（管"怎么写"）→ 输出StrategyBlueprint
└── RAGWriter（管"写什么"）→ 使用RAG事实素材生成文章
```

### 核心能力

| 能力 | 说明 |
|------|------|
| **策略学习** | 从118篇标注文章中学习写作方法论 |
| **动态Few-shot** | 根据话题检索相似案例作为参考 |
| **结构化输出** | 生成JSON格式的策略蓝图，可审核、可调试 |
| **Self-Reflection** | 自动校验策略质量，不合格则重试 |
| **可持续进化** | 案例库可动态扩充，持续优化策略 |

## 三、快速开始

### 3.1 基本使用流程

```python
from rag_writer import create_writer
from rag_writer.skill_agent import SkillAgent, CaseBaseManager

# 1. 创建RAGWriter（已有系统）
writer = create_writer(articles_json="articles.json")

# 2. 创建SkillAgent（复用LLM和Embedding）
skill_agent = writer.create_skill_agent(
    case_base_path="./strategy_cases"
)

# 3. 检索RAG事实素材
rag_context = writer.retrieve_knowledge("大模型定价战", top_k=5)

# 4. 生成策略蓝图
blueprint = skill_agent.generate_blueprint(
    topic="大模型定价战分析",
    rag_context={"knowledge": rag_context},
    content_type="行业分析",
    target_audience="企业高管",
)

# 5. 基于蓝图写作
result = writer.write_from_blueprint(blueprint, knowledge=rag_context)

print(result.article)
```

### 3.2 一键完成（compile_only模式）

```python
# 如果只需要策略蓝图，不立即生成文章
result = skill_agent.write_with_blueprint(
    topic="大模型定价战分析",
    rag_context={"knowledge": rag_context},
    content_type="行业分析",
    target_audience="企业高管",
    compile_only=True,  # 仅返回蓝图和指令，不生成文章
)

# 获取结果
blueprint = result["blueprint"]       # 策略蓝图
instructions = result["instructions"]  # 写作指令
cases = result["case_references"]     # 引用的策略案例

# 可随时基于蓝图生成文章
result = writer.write_from_blueprint(blueprint, knowledge=rag_context)
```

```json

  "annotation": {
    "opening_approach": "数据震撼",
    "opening_effectiveness": 5,
    "structural_pattern": "递进式",
    "section_strategies": [...],
```

## 七、配置参数

### 7.1 SkillAgent配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `temperature` | 0.4 | 策略生成温度，建议0.3~0.5 |
| `case_retrieval_top_k` | 5 | 检索策略案例数量 |
| `max_reflection_iterations` | 2 | 最大校验迭代次数 |
| `min_confidence_threshold` | 0.5 | 置信度阈值 |
| `use_reflection` | True | 是否启用Self-Reflection |

### 7.2 温度参数指南

| 场景 | 建议温度 | 原因 |
|------|----------|------|
| 策略蓝图生成 | 0.3~0.4 | 保证策略稳定、可复现 |
| Self-Reflection校验 | 0.2~0.3 | 校验需要更确定性 |
| 写作执行（RAGWriter） | 0.5~0.6 | 允许一定创造性 |

### 7.3 修改配置

```python
from rag_writer.skill_agent.config import update_skill_config

# 修改配置
update_skill_config(
    temperature=0.35,
    case_retrieval_top_k=3,
    max_reflection_iterations=3
)
```

## 十一、故障排查

### 11.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 策略蓝图解析失败 | LLM输出格式不规范 | 检查LLM响应，增加重试次数 |
| 检索无结果 | Milvus连接失败 | 检查Milvus服务状态 |
| 案例库为空 | 未导入数据 | 运行import_to_casebase.py |
| 置信度过低 | Self-Reflection校验失败 | 检查RAG上下文是否充分 |

### 11.2 调试模式

```python
# 启用详细输出
import logging

# 检查LLM原始响应
response = llm_client.generate_with_system(...)
```

## 附录：Milvus Schema

策略案例库使用独立的`strategy_cases`集合：

| 字段 | 类型 | 说明 |
|------|------|------|
| case_id | VARCHAR(64) | 主键 |
| title | VARCHAR(256) | 文章标题 |
| article_url | VARCHAR(512) | 原文链接 |
| content_type | VARCHAR(64) | 内容类型 |
| audience | VARCHAR(256) | 目标受众 |
| quality_score | FLOAT | 质量评分 |
| tags | ARRAY[VARCHAR] | 标签 |
| embedding | FLOAT_VECTOR(384) | 向量 |
| annotation | VARCHAR(16384) | 策略标注JSON |
| created_at | VARCHAR(32) | 创建时间 |
